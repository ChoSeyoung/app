# PRD – 식재료 관리 기능 (Ingredient Management)

## 1. 개요

### 기능 이름
- Ingredient Management (식재료 관리)

### 목적
- 식재료 기능은 이유식 기록/식단 생성의 기준 데이터 역할을 수행한다.
- 사용자는 식재료 목록 조회, 섭취/반응 상태 확인, 알러지/주의 관리, 관련 기록 조회를 수행할 수 있어야 한다.

## 2. 주요 사용자 스토리

- US-1: 식재료 목록 확인 (먹어본/미도전 확인)
- US-2: 식재료 상태 관리 (NOT_TRIED, TRIED, CAUTION, ALLERGY)
- US-3: 식재료 검색
- US-4: 식재료 추가
- US-5: 식재료 반응 기록 확인
- US-6: 식재료 기반 이유식 기록 확인

## 3. 범위

### 포함
- 식재료 목록 조회/검색
- 식재료 상태 표시/변경
- 식재료 상세 조회
- 식재료 추가
- 식재료 반응 기록 조회
- 식재료 관련 이유식 기록 조회

### 제외 (초기 버전)
- 식재료 영양 정보
- 추천 알고리즘
- 장보기 리스트
- 식재료 자동 생성

## 4. 식재료 상태 정의

| 상태 | 설명 |
| --- | --- |
| `NOT_TRIED` | 아직 먹어보지 않은 식재료 |
| `TRIED` | 먹어본 식재료 |
| `CAUTION` | 주의 반응이 있었던 식재료 |
| `ALLERGY` | 알러지 반응이 있었던 식재료 |

- 상태는 사용자가 수동 변경 가능하다.

## 5. 화면 구성

### 5.1 식재료 목록 화면
- 상단 검색(실시간 필터)
- 상태 필터: 전체 / 먹어봄 / 아직 안먹어봄 / 알러지·주의
- 리스트 항목: 식재료 이름, 상태 아이콘, 첫 시도 날짜(있을 경우)
- 우측 상단: 식재료 추가 버튼
- 항목 탭 시 상세 화면 이동

### 5.2 식재료 상세 화면
- 상단: 식재료 이름, 카테고리
- 상태 정보: 현재 상태, 첫 시도 날짜
- 상태 변경 액션: 먹어봄/주의/알러지/초기화
- 반응 기록 리스트: 날짜, 반응 내용
- 관련 이유식 기록 리스트: 날짜, 함께 먹은 음식

### 5.3 식재료 추가 화면
- 입력: 식재료 이름, 카테고리
- 카테고리 예시: 곡물, 채소, 과일, 단백질, 유제품

## 6. 연동 요구사항

### 기록 기능 연동
- 최근 사용 식재료 노출
- 식재료 검색/선택
- 선택 식재료는 기록 데이터에 포함

### 식단 기능 연동
- 식단 생성 시 식재료 선택 가능
- `TRIED` 우선 표시
- `ALLERGY` 경고 표시

## 7. 데이터 모델

### Ingredient
- `id`
- `name`
- `category`
- `status`
- `firstTriedDate`
- `createdAt`
- `updatedAt`

### IngredientReaction
- `id`
- `ingredientId`
- `date`
- `reactionType`
- `note`

### MealIngredientRelation
- `mealId`
- `ingredientId`

## 8. 상태 변경 규칙

- `NOT_TRIED -> TRIED`: 첫 기록 시 자동 변경 가능
- `TRIED -> CAUTION`: 수동 변경
- `CAUTION -> ALLERGY`: 수동 변경
- `ALLERGY -> TRIED`: 수동 해제 가능

## 9. 예외 처리

- 동일 이름 식재료 중복 생성 방지
- 검색 결과 없음 시 `새 식재료 추가` 액션 제공
- 삭제된 식재료가 기록에 포함된 경우 기록 데이터는 유지

## 10. 향후 확장

- 식재료 영양 정보
- 단계별 추천 식재료
- 식재료 도전 가이드
- 장보기 리스트
- 식단 자동 생성

## 11. 성공 기준

- 식재료 상태를 빠르게 확인 가능
- 기록/식단 생성에서 식재료 선택이 쉬움
- 알러지/주의 반응 관리 가능
