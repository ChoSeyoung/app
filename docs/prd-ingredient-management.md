# PRD – 식재료 관리 기능 (Ingredient Management)

## 1. 개요

### 기능 이름
- Ingredient Management (식재료 관리)

### 목적
- 식재료 기능은 기록, 식단, 레시피가 함께 참조하는 기준 데이터 레이어다.
- 사용자는 식재료 목록을 빠르게 훑고, 먹어본 상태와 주의 식재료를 관리하며, 메모와 즐겨찾기를 통해 다음 선택을 더 쉽게 할 수 있어야 한다.

## 2. 핵심 원칙

- 식재료 마스터는 사용자 추가형이 아니라 `고정형 seed catalog`로 운영한다.
- 식재료 자체 정보와 아기별 상태 정보는 분리해서 관리한다.
- 식재료 화면은 사전(dictionary)보다 `관리 허브`에 가깝게 동작해야 한다.
- 식단/기록과 자연스럽게 이어질 수 있도록 상태, 첫 시도일, 반응 메모를 중심으로 설계한다.

## 3. 주요 사용자 스토리

- US-1: 보호자는 식재료 목록에서 먹어본 재료와 아직 안 먹은 재료를 빠르게 구분할 수 있어야 한다.
- US-2: 보호자는 식재료 상태를 `NOT_TRIED / TRIED / CAUTION / ALLERGY`로 변경할 수 있어야 한다.
- US-3: 보호자는 식재료를 검색하고 바로 상세를 확인할 수 있어야 한다.
- US-4: 보호자는 자주 참고하는 식재료를 즐겨찾기할 수 있어야 한다.
- US-5: 보호자는 식재료 메모를 남기고 다음 식단/기록에 참고할 수 있어야 한다.
- US-6: 보호자는 재시도 시점을 놓치지 않도록 `retrySuggested` 상태를 확인할 수 있어야 한다.

## 4. 범위

### 포함
- 고정형 식재료 마스터 조회
- 식재료 검색
- 상태 필터
- 식재료 상세 바텀시트
- 상태 변경
- 즐겨찾기
- 메모 저장
- 첫 시도일 표시
- 최근 반응 기반 재시도 제안
- 이미지 연결

### 제외
- 사용자 식재료 직접 추가
- 식재료 삭제
- 관련 기록 전체 리스트
- 장보기 리스트
- 영양소 수치 계산

## 5. 식재료 상태 정의

| 상태 | 설명 |
| --- | --- |
| `NOT_TRIED` | 아직 먹어보지 않은 식재료 |
| `TRIED` | 먹어본 식재료 |
| `CAUTION` | 주의 반응이 있었던 식재료 |
| `ALLERGY` | 알러지 반응이 있었던 식재료 |

부가 상태:

- `isFavorite`: 자주 참고하는 식재료 여부
- `latestNote`: 최근 메모
- `retrySuggested`: 주의 반응 이후 일정 시간이 지나 재시도를 검토할 수 있는 상태

## 7. 화면 구성

### 7.1 식재료 목록 화면
- 상단 검색
- 상태 필터: 전체 / 먹어봄 / 미도전 / 주의·알러지 / 재시도
- 2열 카드 그리드
- 카드 정보

  - 식재료 이름
  - 첫 시도일 또는 `미시도`
  - 좋아요 아이콘
  - 주의/알러지 뱃지
  - 메모 한 줄

### 7.2 식재료 상세 바텀시트
- 식재료 이미지
- 이름
- 첫 시도일
- 즐겨찾기 토글
- 상태 변경 액션
- 메모 입력/저장

## 8. 연동 요구사항

### 기록 기능 연동
- 기록 화면의 식재료 검색 결과에서 동일한 마스터 데이터를 사용한다.
- 기록 저장 시 처음 먹은 식재료는 자동으로 `TRIED` 상태로 전환될 수 있다.
- 반응 메모는 식재료 최신 메모와 재시도 판단의 근거가 된다.

### 식단 기능 연동
- 식단 엔진은 식재료 상태와 메타데이터를 읽어 추천 후보를 고른다.
- `ALLERGY`는 항상 제외한다.
- `CAUTION`은 기본 제외한다.
- `minStage`, `observationDays`, `allergyWatch`는 이후 식단 제약의 기준이 된다.

### 레시피 기능 연동
- 식재료는 `relatedRecipeIds`를 통해 레시피와 느슨하게 연결된다.

## 9. 상태 변경 규칙

- `NOT_TRIED -> TRIED`: 기록 생성 또는 수동 변경
- `TRIED -> CAUTION`: 수동 변경
- `CAUTION -> ALLERGY`: 수동 변경
- `ALLERGY -> TRIED`: 수동 해제 가능
- `NOT_TRIED`로 되돌리는 초기화는 허용하되 보호자 의도를 명확히 반영해야 한다.

## 10. 예외 처리

- 고정형 마스터이므로 같은 이름 식재료를 새로 추가하지 않는다.
- 이미지가 없는 식재료는 플레이스홀더를 사용한다.
- 식재료가 아직 미시도인 경우 첫 시도일은 비워둔다.
- 메모가 비어 있어도 식재료 상태는 유지된다.

## 11. 향후 확장

- 식재료 상세에 메타데이터 노출
  - 추천 시작 단계
  - 영양 태그
  - 관련 레시피
- 식단 추천과의 메타데이터 연계 강화
- 식재료 기반 통계
- 단계별 추천 식재료 큐레이션

## 12. 성공 기준

- 사용자가 식재료 상태를 빠르게 확인하고 관리할 수 있다.
- 기록/식단 추천에서 동일한 식재료 마스터를 일관되게 사용한다.
- 주의/알러지 식재료 관리가 명확하게 동작한다.
