# PRD – 식단 기능 (Meal Plan)

## 1. 개요

### 기능 이름
- Meal Plan (식단)

### 목적
- 식단 기능은 아기 상태, 이유식 진행 단계, 식재료 도입 이력, 최근 기록을 기반으로 오늘의 추천과 이번 주 계획을 자동 생성하는 기능이다.
- 사용자는 복잡한 표를 직접 작성하기보다, 현재 아기 기준으로 바로 먹일 수 있는 식단을 확인하고 필요한 경우 조정할 수 있어야 한다.

## 2. 핵심 원칙

- 식단 기준은 `월령` 단독이 아니라 `생후 일수 + 이유식 시작일 + 이유식 주차 + 현재 단계 + 식재료 이력`을 함께 사용한다.
- 추천보다 제약 조건 처리가 우선이다.
- 식단은 설명 가능해야 하며, 보호자가 왜 이런 추천이 나왔는지 이해할 수 있어야 한다.

## 3. 주요 사용자 스토리

- US-1: 보호자는 오늘 먹일 식단을 바로 확인할 수 있어야 한다.
- US-2: 보호자는 이번 주 식단을 한눈에 보고 필요 시 다시 생성할 수 있어야 한다.
- US-3: 보호자는 아직 먹어보지 않은 식재료 도입 시점을 식단에서 확인할 수 있어야 한다.
- US-4: 보호자는 알러지/주의 식재료가 자동 제외된 식단을 받아야 한다.
- US-5: 보호자는 식단에서 바로 `먹였어요`를 눌러 기록 화면으로 전환할 수 있어야 한다.
- US-6: 먹인 결과(먹음, 거부, 메모, 반응)는 다음 식단 추천에 반영되어야 한다.

## 4. 개인화 입력값

### 필수
- 출생일
- 이유식 시작일
- 현재 이유식 단계
- 하루 식사 횟수
- 이유식 방식
- 이미 먹어본 식재료
- 제외 식재료 / 알러지 식재료

### 선택
- 최근 거부 식재료
- 변 상태 / 소화 상태
- 단백질 도입 여부
- 입자감 단계
- 보호자 목표

## 5. 계산 지표

- `calendarAgeDays`: 실제 생후 일수
- `feedingDays`: 이유식 시작 후 일수
- `feedingWeek`: 이유식 n주차
- `feedingStage`: 현재 이유식 단계
- `mealsPerDay`: 현재 하루 식사 횟수

식단 추천의 1차 기준은 `feedingWeek`이다.

## 6. 데이터 레이어

### 6.1 Baby Feeding Profile
- `birthDate`
- `feedingStartDate`
- `feedingStage`
- `mealsPerDay`
- `feedingMethod`
- `proteinStarted`
- `textureLevel`
- `blockedIngredientIds`
- `preferredIngredientIds`
- `dislikedIngredientIds`

### 6.2 Ingredient Metadata
- `id`
- `name`

### 6.3 Derived Ingredient State
- 먹어본 식재료
- 아직 안 먹어본 식재료
- 주의 식재료
- 알러지 식재료
- 최근 메모

## 7. 식단 생성 방식

v1은 `템플릿 + 규칙 기반 치환` 방식을 사용한다.

1. 현재 단계와 식사 횟수에 맞는 기본 슬롯을 생성한다.
2. `TRIED` 식재료를 우선 후보로 사용한다.
3. `NOT_TRIED` 중 현재 단계에서 허용 가능한 식재료를 신규 후보로 선택한다.
4. `CAUTION`, `ALLERGY`, 보호자 제외 식재료는 제외한다.
5. 첫 끼 또는 주간 초반에는 신규 식재료를 1개만 배치한다.
6. 단백질 도입 전이면 단백질은 제외한다.

## 8. 핵심 제약 조건

- 신규 식재료는 한 끼에 1개만 허용
- `ALLERGY` 식재료는 항상 제외
- `CAUTION` 식재료는 기본 제외
- 현재 단계보다 이른 식재료 제외
- 하루 식사 횟수에 맞는 슬롯 수만 생성
- 같은 카테고리 과반 반복 제한
- 단백질 미도입 시 단백질 제외

## 9. 화면 구성

### 9.1 오늘의 추천
- 오늘 추천 식단
- 신규 도입 식재료 여부
- 제외된 식재료 안내
- 이유식 n주차 / 현재 단계 표시

### 9.2 이번 주 계획
- 월~일 주간 식단
- 하루별 끼니 구성
- 신규 식재료 뱃지
- 재생성 가능

### 9.3 식단 상세
- 날짜
- 끼니
- 구성 식재료
- 신규/기존 여부
- 주의 메모
- `먹였어요` 액션

## 10. 기록 기능 연동

- 식단에서 `먹였어요` 선택 시 이유식 기록 화면으로 이동한다.
- 선택한 끼니의 식재료와 시간은 기록 화면에 자동 입력된다.
- 사용자는 섭취량과 반응만 추가 입력하면 된다.

## 11. 초기 버전 범위

### 포함
- 현재 단계/주차 계산
- 오늘 추천 식단
- 주간 식단 자동 생성
- 신규 식재료 표시
- 주의/알러지 식재료 제외
- 식단 -> 기록 전환

### 제외
- AI 추천
- 영양소 자동 계산
- 변 상태/소화 상태 기반 세밀한 조정
- 병원/의료 가이드 자동화

## 12. 성공 기준

- 사용자가 오늘 먹일 식단을 즉시 확인할 수 있다.
- 식재료 상태와 연동된 주간 식단이 생성된다.
- `먹였어요`를 통해 기록 화면으로 자연스럽게 전환된다.
