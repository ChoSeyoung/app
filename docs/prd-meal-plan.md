# PRD – 식단 기능 (Meal Plan)

## 1. 개요

### 기능 이름
- Meal Plan (식단)

### 목적
- 식단 기능은 아기 프로필, 이유식 진행 상태, 식재료 이력, 최근 기록을 바탕으로 `오늘 먹일 식단`과 `이번 주 계획`을 제안하는 기능이다.
- 보호자는 복잡한 표를 직접 짜기보다, 현재 아기 기준으로 바로 실행 가능한 추천을 확인하고 기록으로 이어갈 수 있어야 한다.

## 2. 핵심 원칙

- 기준은 `월령 단독`이 아니라 `출생일 + 이유식 시작일 + 이유식 주차 + 현재 단계 + 식재료 상태`를 함께 사용한다.
- 추천 자체보다 `제약 조건 처리`가 우선이다.
- 추천 이유는 화면에서 설명 가능해야 한다.
- 식단은 기록 기능으로 바로 연결돼야 한다.

## 3. 주요 사용자 스토리

- US-1: 보호자는 오늘 먹일 식단을 바로 확인할 수 있어야 한다.
- US-2: 보호자는 이번 주 계획을 날짜별로 볼 수 있어야 한다.
- US-3: 보호자는 추천 이유를 이해할 수 있어야 한다.
- US-4: 보호자는 아직 입력하지 않은 식단 기준 정보를 화면에서 바로 저장할 수 있어야 한다.
- US-5: 보호자는 식단에서 `먹였어요`를 눌러 기록 추가 화면으로 자연스럽게 이동할 수 있어야 한다.
- US-6: 최근 반응과 거부 이력은 다음 식단 추천에 반영되어야 한다.

## 4. 개인화 입력값

### 필수
- 출생일
- 이유식 시작일
- 현재 이유식 단계
- 하루 식사 횟수
- 이유식 방식

### 선택
- 단백질 도입 여부
- 입자감 단계
- 보호자 목표
- 선호 식재료
- 비선호 식재료
- 제외 식재료
- 최근 거부 식재료

## 5. 계산 지표

- `calendarAgeDays`: 생후 일수
- `feedingDays`: 이유식 시작 후 일수
- `feedingWeek`: 이유식 n주차
- `feedingStage`: 현재 단계
- `mealsPerDay`: 현재 식사 횟수

추천의 기준축은 `feedingWeek`다.

- 최근 메모 힌트

## 7. 식단 생성 방식

현재 버전은 `규칙 기반 추천 엔진`을 사용한다.

1. 현재 단계와 식사 횟수에 맞는 슬롯을 만든다.
2. `TRIED` 식재료를 우선 후보로 사용한다.
3. 필요할 경우 `NOT_TRIED` 식재료 중 허용 가능한 재료를 신규 후보로 선택한다.
4. `ALLERGY`, `CAUTION`, 차단 식재료, 최근 거부 식재료는 우선 제외한다.
5. 관찰 중 식재료가 있으면 그 재료를 우선 유지한다.
6. 즐겨찾기 식재료는 후보 우선순위를 올린다.
7. 단백질 도입 전이면 단백질을 넣지 않는다.

## 8. 핵심 제약 조건

- 신규 식재료는 같은 날/첫 끼 기준 1개만 허용
- `ALLERGY` 식재료는 항상 제외
- `CAUTION` 식재료는 기본 제외
- 현재 단계보다 이른 식재료 제외
- 최근 거부 식재료 우선 제외
- 식사 횟수에 맞는 슬롯 수만 생성
- 관찰 중 식재료는 다른 신규 식재료보다 우선

## 9. 화면 구성

### 9.1 아이 정보
- 생후 일수
- 이유식 주차
- 단계
- 식사 수
- 방식

### 9.2 오늘 추천
- 오늘 첫 끼 추천
- 끼니별 식재료 구성
- 신규 식재료 포함 여부
- 메모 타입(관찰 / 주의 제외)

### 9.3 추천 이유
- 현재 추천이 왜 나왔는지 요약 카드
- 관찰 우선 / 최근 거부 제외 / 즐겨찾기 반영 등 설명

### 9.4 주간 계획
- 주간 달력
- 선택 날짜 식단
- 주간 계획 목록

### 9.5 기준 입력 카드
- 이유식 시작일 미입력 시 바로 저장할 수 있는 카드 제공

## 10. 기록 기능 연동

- 식단에서 `먹였어요` 선택 시 기록 추가 페이지로 이동한다.
- 선택한 식단의 식재료와 시간은 기록 화면에 자동 입력된다.
- 사용자는 섭취량과 반응만 보완하면 된다.

## 11. 현재 버전 포함 범위

### 포함
- 현재 단계/주차 계산
- 오늘 추천 식단
- 주간 계획
- 추천 이유 요약
- 식단 기준 입력 저장
- 기록 화면 전환
- 최근 반응/거부/관찰 상태 반영

### 제외
- AI 추천
- 영양소 정밀 계산
- 의학적 판단 자동화
- 보호자 직접 식단 편집기

## 12. 향후 확장

- `minStage`, `allergyWatch`, `observationDays`를 엔진 제약에 더 깊게 반영
- 관련 레시피 기반 식단 추천
- 추천 식단 교체/세부 조정
- 주간 식단 회고/통계

## 13. 성공 기준

- 보호자가 오늘 먹일 식단을 즉시 확인할 수 있다.
- 식재료 상태와 최근 기록이 실제 추천에 반영된다.
- `먹였어요`를 통해 기록 화면으로 자연스럽게 이어진다.
- 추천 이유가 화면에서 이해 가능하다.
