# Weaning Diary Language Guide v1

목적: 앱 내 모든 문구를 같은 말투/정보 구조로 통일해, 보호자가 빠르게 이해하고 행동하게 만든다.

## 1) 기본 원칙

- 기본 언어: 한국어(`ko`)
- 톤: 차분하고 다정하게, 과장 없이 명확하게
- 기본 인상: 귀엽고 포근하지만 유치하지 않게 유지
- 관점: 보호자 중심(사용자를 탓하지 않음)
- 길이: 한 문장 20자 내외 권장, 길면 두 문장으로 분리

## 2) 문구 구조

### 화면 상단(Title/Sub)
- Title: "무엇을 해야 하는지"를 명사형/동사형으로 짧게
- Sub: 왜 필요한지 1문장으로 설명

예시
- Title: `아기 프로필 설정`
- Sub: `입력한 정보로 홈 화면을 맞춤 제공해요.`

### 입력 라벨
- 라벨은 구체적이고 선택지를 포함
- 예: `아기 이름 / 태명 / 별명`

### 날짜 표기
- 사용자에게 보여주는 날짜는 기본적으로 `YYYY.MM.DD` 형식으로 통일한다.
- 요일이 함께 필요할 때는 `YYYY.MM.DD · 화`처럼 같은 형식을 유지한 채 뒤에 붙인다.
- 입력 포맷은 화면 목적에 따라 `YYYYMMDD` 또는 `YYYY-MM-DD`를 사용할 수 있지만, 저장 후 표시 형식은 통일한다.

### 버튼 라벨
- 즉시 행동형 동사 사용
- 예: `다음`, `프로필 저장`, `사진 업로드`

### 안내 문구
- 딱딱한 관리 도구 문체보다 보호자를 부드럽게 안내하는 문장을 우선한다.
- 예: `오늘 이유식 기록이 아직 없어요.`
- 예: `가볍게 시작해도 괜찮아요.`

### 경고/확인 메시지
- 1문장으로 사실 전달 -> 1문장으로 확인 요청
- 예: `입력한 생년월일 기준으로 24개월 이상입니다. 잘 입력했는지 확인해주세요.`

## 3) 금지/지양 표현

- 내부 용어: `온보딩`, `아바타`, `게이트` 같은 개발자 용어
- 추상 표현: `진행해보세요`, `완료해주세요` 같은 모호한 문구
- 불안 유발: 과한 경고 어조, 느낌표 남발
- 업무용/관리자 톤: 지나치게 건조한 문장, 표어 같은 문장

## 4) 용어 사전

- User -> `보호자`
- Baby profile -> `아기 프로필`
- Birth date -> `생년월일`
- Home avatar image -> `프로필 사진`
- Step/Onboarding -> `초기 설정`

## 5) i18n 운영 규칙

- 모든 문구는 `constants/i18n.ts`에서만 관리
- 컴포넌트 하드코딩 금지
- 새 화면 추가 시 `ko` 먼저 작성, `en` 동시 추가
- key는 기능 단위로 네임스페이스 유지
  - 예: `home.profileForm.title`

## 6) 카피 체크리스트

1. 한 번 읽고 바로 행동이 떠오르는가?
2. 보호자에게 책임을 전가하는 어조가 없는가?
3. 기술 용어가 사용자 문구에 노출되지 않는가?
4. 같은 의미를 화면마다 다르게 말하고 있지 않은가?
5. i18n key로만 관리되고 있는가?
