# Weaning Diary Sprint 1 Backlog

목표: 핵심 사용자 흐름(기록 입력 -> 조회/필터)을 작동 가능한 수준으로 완성한다.

기간: 1주 (초안)

## 운영 규칙

- 우선순위: P0 > P1 > P2
- 상태: Todo / In Progress / Done
- 모든 이슈는 완료 시 PR에 테스트 결과(수동 또는 자동)를 남긴다.

## Sprint 1 이슈 목록

### WD-001 (P0) 기록 도메인 데이터 모델 정의

- 상태: Todo
- 목표: 식단(계획)과 기록(실제)을 분리한 도메인 타입을 정의한다.
- 작업:
  - `features/records/model.ts`에 도메인 타입 정의
  - `PlannedMeal`(계획), `MealRecord`(실제), `RecordDraft`(초안) 스키마 확정
  - enum/union으로 반응 타입(`skin`, `digestive`, `stool`, `sleep`) 정의
  - 생성/수정 시각 필드와 ID 전략(UUID 또는 timestamp+random) 확정
- 완료조건 (DoD):
  - 타입만으로 계획/실제 차이를 명확히 표현할 수 있다.
  - 예시 케이스(계획 단호박 50g, 실제 단호박 20g+고구마 추가, 반응 분리)가 모델에서 표현 가능하다.
  - 잘못된 반응 타입 입력이 타입 단계에서 차단된다.
- 의존성: 없음

### WD-002 (P0) 로컬 저장소 레이어 구현

- 상태: Todo
- 목표: 기록 데이터를 앱 재실행 후에도 유지한다.
- 작업:
  - 저장소 구현 방식 결정(AsyncStorage 또는 SQLite)
  - CRUD 인터페이스 정의: `createEntry`, `listEntries`, `getEntry`, `updateEntry`, `deleteEntry`
  - 기본 정렬(최신순)과 날짜 필터 API 제공
  - 저장 실패/파싱 실패 에러 처리
- 완료조건 (DoD):
  - 앱 재실행 후 데이터가 유지된다.
  - 날짜 기준 목록 조회가 정상 동작한다.
- 의존성: WD-001

### WD-003 (P0) 기록 입력 화면 MVP

- 상태: Todo
- 목표: 30초 내 1건 기록 가능한 입력 경험을 제공한다.
- 작업:
  - 필수 입력: 날짜, 시간, 식재료, 섭취량, 반응(선택), 메모(선택)
  - 빠른 입력 UX: 최근 식재료 자동 제안, 기본 시간값 현재시각
  - 저장 버튼/저장 중 상태/성공-실패 피드백 구현
  - 필수값 누락 검증 메시지 구현
- 완료조건 (DoD):
  - 최소 입력값으로 새 기록 생성 가능
  - 저장 후 타임라인에 즉시 반영
  - 오류 발생 시 사용자에게 이유가 표시
- 의존성: WD-001, WD-002
