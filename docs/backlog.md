# Weaning Diary Product Backlog

기준 날짜: 2026-03-14

목표: 현재 구현된 `홈 / 식단 / 기록 / 식재료 / 가이드 / 더보기`를 실제 반복 사용 가능한 서비스 단계로 끌어올린다.

## 운영 규칙

- 우선순위: `P0 > P1 > P2`
- 상태: `Todo / In Progress / Done`

- UI 변경은 반드시 `docs/design-system.md` 기준으로 맞춘다.
- 모든 이슈는 완료 시 `npm run lint` 기준 검증을 남긴다.

## 현재 상태 요약

- 이유식 시작하기 가이드 / 체크리스트 / FAQ / 책갈피
- 더보기 / 알림 설정 UI / 문서 페이지

### 현재 부족한 핵심
- 실제 로컬 알림 발송
- 식단 개인화 입력 확장
- 기록 결과의 식단 추천 반영 강화
- 데이터 백업 / 복원
- 홈 행동 유도 강화
- 통계 / 회고 화면

## P0

### WD-201 (P0) 로컬 알림 기능 구현

- 상태: `Done`
- 목표: 알림 설정 화면의 토글이 실제 로컬 알림 스케줄로 이어지게 한다.
- 작업:
  - Expo 로컬 알림 도입 및 권한 요청
  - 아침 식단 알림 스케줄링
  - 기록 리마인더 알림 스케줄링
  - 조용한 시간 적용
  - 설정 변경 시 기존 스케줄 정리 후 재등록
- 완료 조건:
  - 설정 토글 변경 후 앱 재실행에도 스케줄이 유지된다.
  - 끈 알림은 실제로 발송되지 않는다.
  - 조용한 시간 중에는 알림이 예약되지 않거나 지연된다.

### WD-202 (P0) 식단 개인화 입력 확장

- 상태: `Done`
- 목표: 식단 추천에 필요한 실제 사용자 상태를 더 받는다.
- 작업:
  - 프로필/식단 진입에서 단계, 식사 수, 방식 외 입력 확장
  - 단백질 도입 여부
  - 입자감 단계
  - 제외 식재료
  - 비선호 식재료
  - 최근 거부 식재료
- 완료 조건:
  - 식단 추천에 필요한 핵심 입력이 프로필에 저장된다.
  - 입력값이 식단 추천 UI와 엔진 양쪽에 반영된다.

### WD-203 (P0) 기록 기반 식단 제약 강화

- 상태: `Done`
- 목표: 기록 결과가 다음 식단 추천에 더 직접적으로 반영된다.
- 작업:
  - 최근 거부 식재료 제외 또는 우선순위 하향
  - 최근 발진/토함 식재료 자동 주의 처리
  - 최근 기록 메모를 식단 힌트에 반영
  - 신규 식재료 관찰 중이면 추천 폭 자동 축소
- 완료 조건:
  - 기록 후 식단 추천이 눈에 띄게 바뀌는 케이스가 존재한다.
  - 위험 식재료가 추천에 그대로 남지 않는다.
